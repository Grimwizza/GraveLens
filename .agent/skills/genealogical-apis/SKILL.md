### 1. API Endpoints & Request Specifications

* **OpenStreetMap (OSM) Overpass API**
  * **Endpoint**: `https://overpass-api.de/api/interpreter`
  * **Usage**: Fetch cemetery features within an `around` boundary (typically `around:800,lat,lng`).
  * **Tags to Match**: `landuse="cemetery"`, `amenity="grave_yard"`.
  * **Property Mapping**: `opening_hours`, `phone` / `contact:phone`, `website` / `contact:website` / `url`, `denomination` / `religion`, `start_date`, `wikipedia`, `wikidata`.

* **NARA Catalog API v2**
  * **Endpoint**: `https://catalog.archives.gov/api/v2`
  * **Usage**: Elasticsearch-backed search using full names and military keywords. Always use `DEMO_KEY` as fallback key.
  * **Important**: Indexes finding aids and series, not individuals.
  * **Target Record Groups (RGs)**:
    * `RG 15`: Veterans Administration pension files.
    * `RG 24`: Bureau of Naval Personnel.
    * `RG 94`: Adjutant General's Office (Army, pre-WWII).
    * `RG 120`: American Expeditionary Forces (WWI).
    * `RG 407`: Adjutant General's Office (WWII).

* **FamilySearch Platform API**
  * **Endpoint**: `https://api.familysearch.org/platform/records/search`
  * **Headers**: `Accept: application/x-gedcomx-atom+json` or `application/json`.
  * **Collections**:
    * SSDI: `2437639`
    * Civil War Pension Index: `1932460`
    * WWI Draft Registration Cards: `1968530`
  * **Parameters**: `q.givenName`, `q.surname`, `q.birthLikeDate.from`, `q.birthLikeDate.to`.

* **Bureau of Land Management (BLM) GLO**
  * **Usage**: Land patents by state and name.

* **Wikidata SPARQL API**
  * **Endpoint**: `https://query.wikidata.org/sparql`
  * **Usage**: Retrieve notable figures and historical landmarks in map bounding boxes.

### 2. Search Phonetics (`src/lib/phonetic.ts`)
* All name-based external searches must utilize surname phonetic codes generated by standard Soundex (4-character NARA-compatible) and Double Metaphone to widen query queries and capture spelling mutations (e.g. Schmitt/Smith, Schmidt/Smyth).

### 3. Cross-Validation Matrix
Cross-reference overlapping data from separate APIs to assess matching confidence:
1. **Name Matching**: Verify OCR name against SSDI / Census names. Warn if edit distance > 3 chars.
2. **Birth/Death Years**: Compare gravestone birth/death year against enlistment record/census age/SSDI birth year. Flag discrepancy > 2 years.
3. **Location/Geography**: Match Cemetery GPS state against BLM patent state, SSDI last-residence state, or census county. Note migration pattern if states differ.
4. **Foreign Language / Inscription**: For non-English inscriptions (e.g., German, Polish), prioritize Castle Garden/Ellis Island databases for corresponding nationalities.
5. **Land patents & Census**: Confirm settlement pattern if land patent location matches census county or probate records.
